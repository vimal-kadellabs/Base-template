## State Management Overview

### Global State (AuthContext)
- Current user
- Authentication status
- Session data
- Accessible from any component via useAuth()

### Local State (Component-specific)
- Form inputs
- UI state (modals, dropdowns)
- Temporary data

### Persistent State (localStorage)
- Users array
- Configuration object
- Session object
- Automatically synced by hooks

---

## Key Benefits of This Architecture

1. **Unidirectional Data Flow**
   - Clear path from user action to state update
   - Easy to trace and debug

2. **Separation of Concerns**
   - Components handle UI
   - Hooks handle state & logic
   - Utils handle operations
   - Constants provide config

3. **Reusability**
   - Hooks can be used in any component
   - Utils can be used in any hook
   - No code duplication

4. **Testability**
   - Each layer can be tested independently
   - Pure functions in utils layer
   - Hooks can be tested with React Testing Library

5. **Maintainability**
   - Changes in one layer don't affect others
   - Easy to locate code
   - Clear responsibility boundaries

6. **Scalability**
   - Easy to add new hooks
   - Easy to add new utilities
   - Easy to extend functionality
